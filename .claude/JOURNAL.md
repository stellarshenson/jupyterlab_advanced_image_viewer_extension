3. **Task - Fix make install build break** (v0.1.1): Fixed `make install` failing in the labextension production webpack build, diagnosed via the `jupyterlab-extension` skill<br>
   **Result**: `jlpm run build:prod` crashed with `TypeError: Cannot read properties of undefined (reading 'trim')` in `license-webpack-plugin`'s `getActualFilename`. Root cause is the documented webpack >= 5.106.1 incompatibility: webpack changed its module-federation share identifier from `name@ver = req` to `name@ver|req`, so the plugin's `split('=')[1].trim()` reads `undefined`. The plugin is unmaintained with no forward fix, so pinned `webpack` to `5.106.0` in both `resolutions` (jlpm) and `overrides` (npm). Since the host runs Node 24.16.0 and `make build` uses the `python -m build` isolated path, also pinned `chalk` to `4.1.2` (both blocks) to pre-empt the Node-24 `chalk@2.4.2` `module.exports.default` setter crash in `duplicate-package-checker-webpack-plugin`. Wiped `node_modules`/lockfiles and reran: `webpack 5.106.0 compiled successfully`, wheel built, `jupyter labextension list` shows v0.1.1 enabled OK. New `package-lock.json` and `yarn.lock` must be committed together with `package.json`.

4. **Task [Extended] - Fix image fit, zoom-out, nav; verify via Playwright** (v0.4.6): Reworked the viewer after the user reported broken fit (tiny/fragment/bottom-aligned images, worst on viewBox-only SVGs) and validated against acceptance criteria<br>
   **Result**: Root cause found by live Playwright DOM capture with the extension uninstalled: the stock viewer fits purely via CSS (`object-fit: contain; max-width/height: 100%`, top-left, no transform), but `ViewerController` computed scale/translate from `img.naturalWidth/Height` and wrote its own transform on top - double-transforming. SVGs were worst because a `viewBox="0 0 800 60"` image reports a bogus `naturalWidth=300,Height=23`, so the old `min(host/natural)` math produced `scale(0.66) translate(0,178px)` (131px, shoved down). Rewrote `src/controller.ts` to treat identity transform as the fit baseline (never reads naturalWidth), layering only relative zoom (`s=1` is fit) and pan; cursor-anchored zoom about host centre; removed the fit-scale floor so zoom-out is unlimited (`MIN_SCALE=0.05`); pan bounded by `(s-1)/2*host`. Per user, made nav arrows-only (hardcoded Left/Right, dropped `prevKey`/`nextKey` from `index.ts` + `schema/plugin.json`); nav opens next image and disposes the previous widget so one tab advances. Added the activation `console.log` back (Galata test) and fixed CI for frontend-only: `build.yml` drops the server-extension `grep` + `pytest` and adds badge `ignore_links`; `check-release.yml` gets `steps_to_skip: build-changelog` + `RH_SINCE_LAST_STABLE`. Wrote `docs/image_fit_reference.md` and `docs/acceptance_criteria.md` (AC1-A7 fit, B zoom-in, C unlimited zoom-out, D reset, E pan, F single-tab arrow nav, G/H) with stock reference screenshots. Set version to 0.4.6. Playwright verification on the reference SVG/PNG files: all baselines render identity-transform, full-fit, matching stock numbers exactly (growth SVG 918x69, image-2 314x306 natural, warping 918x423); zoom-in 1.0->1.61, zoom-out to 0.239 below fit, reset to identity, pan bounded, ArrowRight advances within a single tab.
